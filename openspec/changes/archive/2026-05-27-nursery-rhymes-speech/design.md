## Context

`h5/jingdian-tongyao.html` 是一个自包含的静态 HTML 页面，包含 60 首经典童谣，分为 4 个分类（国风童谣、经典童谣、校园金曲、自然亲情），每个童谣以 `.poem-card` 卡片形式展示。页面使用内联 CSS/JS，零外部依赖，部署在 GitHub Pages。

当前每张卡片包含：标题、体裁标签、年代/来源、童谣正文、标签。无任何音频/语音交互能力。

## Goals / Non-Goals

**Goals:**
- 在每张童谣卡片上增加一个朗读按钮，使用 Web Speech API 朗读童谣文本
- 朗读按钮与现有卡片视觉风格统一，不破坏现有布局
- 提供播放/停止状态切换，同一时刻仅一个朗读任务
- 自动检测并选择中文普通话语音（`zh-CN`）
- 深色模式下朗读按钮样式正确

**Non-Goals:**
- 不实现语音合成（TTS 合成音频文件），仅使用浏览器原生语音
- 不添加语速/音量/音调的全局控制面板（保持简洁）
- 不实现逐行高亮朗读进度（复杂度过高）
- 不修改其他页面的功能
- 不引入任何第三方库或 API 密钥

## Decisions

### 1. 使用 Web Speech API（SpeechSynthesis）而非第三方 TTS 服务

**选择**：浏览器原生 `window.speechSynthesis`

**备选方案**：
- 第三方云 TTS（如 Azure Cognitive Services、AWS Polly）：需要 API 密钥、有调用费用、增加依赖
- 预录音频文件：60 首童谣需要录制和托管音频文件，维护成本高

**理由**：项目是零依赖纯静态站点，Web Speech API 零成本、零配置、所有现代浏览器支持，且中文语音质量在 Chrome/Safari 上已足够好。唯一缺点是语音质量因浏览器和操作系统而异，但对少儿学习场景完全够用。

### 2. 朗读按钮位置：卡片标题行右侧

**选择**：在 `.poem-card-header` 的 `poem-form` 标签旁添加朗读按钮

**备选方案**：
- 卡片底部独立行：增加卡片高度，视觉不紧凑
- 正文区域浮动按钮：干扰阅读体验

**理由**：标题行右侧空间充足，与现有 `.poem-form` 标签并列自然，不增加卡片高度。

### 3. 单例朗读（同一时刻仅一个朗读任务）

**选择**：新播放自动停止前一个朗读

**理由**：多首同时朗读会造成语音混乱，少儿用户难以分辨。单例模式交互更清晰。

### 4. 按钮状态：播放 ↔ 停止切换

**选择**：同一按钮切换播放/停止，通过 CSS class 区分状态

**理由**：减少按钮数量，交互直觉——点击即播放/停止。图标使用内联 SVG（扬声器/停止），无需引入图标库。

## Risks / Trade-offs

- **[语音质量不一致]** 不同浏览器/OS 的中文语音引擎质量差异大（Chrome 最好，Firefox 依赖系统语音） → 无法控制，可接受——大部分用户使用 Chrome/Safari
- **[语音引擎加载延迟]** 首次使用时浏览器可能需要下载语音包（几十 MB） → 在页面加载时预热 `speechSynthesis.getVoices()`，首次点击时提示用户
- **[iOS Safari 限制]** iOS Safari 需要用户交互才能触发语音，且 `speechSynthesis.speaking` 状态可能不准确 → 通过按钮状态管理而非依赖 API 状态
- **[长文本朗读]** 60 首童谣文本均较短（2-4 行），不存在长文本朗读问题 → 无需处理
